You Don't Know Rämpf — Netzwerk-Modus

Was wird gebraucht?

  • 1 Laptop/PC — Präsentation (grosser Bildschirm / Beamer)
  • 1 Tablet/Handy — Moderator-Steuerung
  • 2 Tablets/Handys — je eins pro Spieler
+---------------------------+-----------------+ | | MODERATOR | | PRÄSENTATION | Steuerung, | | (Beamer / grosser | richtige | | Bildschirm) | Antwort | | +-----------------+ | Titel, Fragen, | SPIELER SPIELER| | Punkte, Animationen | 1 2 | +---------------------------+-----------------+

Spiel starten (Schritt für Schritt)

1 Host starten presentation

Öffne die Spiel-URL mit dem Parameter #role=presentation auf dem Laptop/PC, der am Beamer hängt. Das Spiel startet als Host und zeigt oben rechts einen Join-Code an.

Beispiel: https://<deine-url>/Game.qml#role=presentation
2 Moderator verbinden mod

Öffne die URL mit #role=mod auf dem Moderator-Tablet. Du kannst den Code entweder direkt in der URL mitgeben oder auf dem Join-Screen eintippen.

Mit Code in der URL: ...#role=mod&session=XXXX
Oder: Code manuell eingeben auf dem Join-Screen
3 Spieler 1 verbinden basti

Spieler 1 öffnet die URL mit #role=basti und gibt denselben Join-Code ein.

Mit Code in der URL: ...#role=basti&session=XXXX
4 Spieler 2 verbinden dacrowd

Spieler 2 öffnet die URL mit #role=dacrowd und gibt denselben Join-Code ein.

Mit Code in der URL: ...#role=dacrowd&session=XXXX
5 Losspielen!

Sobald alle verbunden sind, steuert der Moderator das Spiel: Runde starten, Antwort aufdecken, weiter. Die Spieler antworten über ihre Buzzer-Buttons auf dem eigenen Gerät.

Verbindung verloren?

Kein Problem — das Spiel versucht automatisch alle 3 Sekunden sich neu zu verbinden. Nach dem Reconnect wird die Rolle erneut registriert und der aktuelle Spielstand synchronisiert.

Standalone-Modus

Tipp: Ohne #role=... Parameter startet das Spiel im Standalone-Modus — alle Views (Präsentation, Moderator, Spieler) werden nebeneinander in einem einzigen Fenster angezeigt. Praktisch zum Testen oder wenn nur ein Bildschirm verfügbar ist.
